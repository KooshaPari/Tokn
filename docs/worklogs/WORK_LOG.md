# Tokn Worklog Index

This file is the repo-local worklog ledger entrypoint for automation and agent
handoffs. Keep active project status in `docs/WORKLOG.md`; keep dated
implementation history in `docs/worklog/`.

## Canonical Surfaces

- `docs/WORKLOG.md` - current project status, open concerns, and next actions.
- `docs/worklog/INDEX.md` - dated historical worklog chronology.
- `docs/worklogs/README.md` - worklog category policy and entry format.

## Current Routing

- New active status updates should amend `docs/WORKLOG.md`.
- New dated entries should be added under `docs/worklog/` and linked from
  `docs/worklog/INDEX.md`.
- Governance/category policy changes should amend `docs/worklogs/README.md`.
